# Domain Objects Description

This document provides a detailed description of the primary actors and core business entities within the BatDongScam system, focusing on their roles, permissions, and data responsibilities.

| # | Object Name | Object Description |
|---|---|---|
| 1 | System Administrator | Users in this group have permission to manage user accounts, oversee global property listings, resolve violation reports, monitor system audit logs, and approve the release of escrowed funds. |
| 2 | Property Owner | Users in this group have permission to create and manage property listings, request AI-driven valuations, purchase promotion packages, and view engagement insights for their properties. |
| 3 | Sale Agent | Users in this group have permission to manage assigned property listings, confirm or cancel viewing appointments, communicate with customers via real-time chat, and subscribe to premium plans. |
| 4 | Customer | Users in this group have permission to search and filter properties, manage personal wishlists, book viewing appointments, sign digital contracts, and provide ratings and feedback for agents. |
| 5 | Registered User | Users in this group have permission to maintain their personal profile, participate in the referral program, report suspicious activity, and receive system notifications. |
| 6 | Guest | Users in this group have permission to browse public property listings, view basic location data, and register for a new account on the platform. |
| 7 | Property Listing | The object contains the information of real estate assets, including physical specifications, pricing, location data, and media. Created when an Owner or Administrator submits a new listing. |
| 8 | Contract | The object contains the information of legal agreements between parties, including terms for rental, deposit, or purchase transactions. Created when a transaction process is initiated. |
| 9 | Appointment | The object contains the information of scheduled property viewings, including the customer, the assigned agent, and the meeting time. Created when a Customer requests a viewing. |
| 10 | Payment | The object contains the information of financial transactions, including amounts, statuses, and external gateway references. Created when a user pays for a contract, subscription, or promotion. |
| 11 | Violation Report | The object contains the information of reported suspicious behavior or content for administrative review. Created when a user flags a listing or account. |
| 12 | Review | The object contains the information of feedback and numerical ratings provided by customers. Created after a completed appointment or contract to evaluate an agent's performance. |
| 13 | Notification | The object contains the information of automated alerts and messages dispatched to users. Created by system events such as appointment confirmations or payment successes. |
| 14 | Subscription Plan | The object contains the information of premium tiers that define listing capacities and features for agents. Created when a Sale Agent purchases a service tier. |
| 15 | AI Valuation | The object contains the information of estimated market values generated by machine learning algorithms based on property features. Created when an Owner requests an automated price estimation. |
| 16 | Identification Document | The object contains the information of legal identity or land titles used for verification and automated form completion. Created when a user uploads images for verification or contract generation. |
