# AD-011: Use Micrometer Tracing for End-to-End Observability

*   **Status:** Proposed
*   **Context:** A single user request now flows from the API Gateway to the Macroservice, potentially publishes an event, and triggers logic in the Legacy Monolith. Debugging performance bottlenecks or errors across these boundaries is nearly impossible with standard logs.
*   **Decision:** Implement **Micrometer Tracing** (formerly Spring Cloud Sleuth) and export trace data to a central collector (e.g., Zipkin or Jaeger).
*   **Consequences (Positive):**
    *   **Visibility:** Provides a visual "Trace ID" for every request, showing exactly how long each hop (Gateway -> Service -> DB) took.
    *   **Rapid Debugging:** Allows developers to see the entire lifecycle of a failed request across all microservices in a single view.
*   **Consequences (Negative/Risks & Mitigation):**
    *   **Performance Overhead:** Tracing adds a small amount of latency and log volume. *Mitigation:* Use sampling (e.g., trace only 10% of requests in production) to minimize impact.
