# AD-008: Establish a Shared Kernel for DTOs, Events, and Exceptions

*   **Status:** Accepted
*   **Context:** Our microservices architecture requires consistent communication structures. If the Property service and the Search service use different definitions for a `PropertyCreatedEvent`, integration becomes brittle.
*   **Decision:** We utilize the `bds-common` module as a "Shared Kernel." This module contains universal DTOs, Event schemas, and base Exception classes (e.g., `BadRequestException`) that are reused across all microservices and the legacy monolith.
*   **Consequences (Positive):**
    *   **Type Safety:** Ensures that producers and consumers of events/APIs are always in sync at the code level.
    *   **Reduced Duplication:** Prevents rewriting common utility logic (like JWT parsing or base exception handling) in every service.
*   **Consequences (Negative/Risks & Mitigation):**
    *   **Coupling:** Changes to `bds-common` require all services to be recompiled/redeployed. *Mitigation:* Only include truly stable, cross-cutting concerns in the Shared Kernel. Domain-specific logic remains within its respective service.
