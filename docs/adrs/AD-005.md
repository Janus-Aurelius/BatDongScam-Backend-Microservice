# AD-005: Adopt Polyglot Persistence Strategy

*   **Status:** Accepted
*   **Context:** The `legacy/` system already contains both `spring-boot-starter-data-jpa` and `spring-boot-starter-data-mongodb`. The relational data (PostgreSQL) handles complex contracts, while the document data (MongoDB) handles unstructured logs and reports.
*   **Decision:** Formally adopt a Polyglot Persistence strategy where the storage technology is chosen based on the specific domain's data model and access patterns.
*   **Consequences (Positive):**
    *   **Efficiency:** Uses PostgreSQL for ACID-compliant transactions (Contracts/Payments) and MongoDB for high-throughput, flexible schema data (Search/Reports).
    *   **Optimization:** Allows us to leverage specialized features like PostGIS for map-based searches in the future.
*   **Consequences (Negative/Risks & Mitigation):**
    *   **Operational Overhead:** Managing two distinct database systems. *Mitigation:* Maintain strict boundary ownership; a single microservice should "own" its specific database instance to prevent schema cross-talk.
