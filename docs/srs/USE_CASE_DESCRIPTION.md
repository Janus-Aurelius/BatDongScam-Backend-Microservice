# Description of Use Cases

| Use Case Name | Definition |
| :--- | :--- |
| **UC1: Sign Up** | This use case describes the process by which a user creates a new account in the system. |
| **UC2: Sign In** | This use case describes the process by which a user logs into the system. |
| **UC3: Create Property Listing** | This use case allows Owners or Admins to create a property listing with images and documents. |
| **UC4: Search Properties** | This use case describes the process by which a user searches for properties using various filters such as location, price, and property type. |
| **UC5: View Property Details** | This use case describes the process by which a user views the comprehensive information of a specific property. |
| **UC6: Book Appointment** | This use case allows a user to request a viewing appointment for a specific property. |
| **UC7: Manage Appointment** | This use case describes how an authorized user can confirm an appointment (by assigning a Sales Agent) or cancel a viewing request. |
| **UC8: Create Rental Contract Draft** | This use case allows an Admin or Agent to generate a draft for a rental contract using property and party information. |
| **UC9: Sign Deposit Contract** | This use case describes the process by which a customer signs a draft deposit contract to reserve a property. |
| **UC10: Initialize Payment** | This use case describes the process by which a user initiates a payment for a contract or service via an external payment gateway. |
| **UC11: Handle Payment Webhook** | This use case describes how the system processes asynchronous payment confirmation notifications from external gateways. |
| **UC12: Report Violation** | This use case allows a user to report suspicious property listings or user behavior to the administration. |
| **UC13: Review Violations** | This use case describes how an Administrator reviews a reported violation and applies disciplinary actions if necessary. |
| **UC14: Send Push Notification** | This use case describes the process by which the system sends real-time alerts to users based on system events. |
| **UC15: Send Email Alert** | This use case describes the process by which the system sends automated email notifications and documents to users. |
| **UC16: Generate Sales Report** | This use case describes how an Administrator can generate financial and transaction reports for a specific year. |
| **UC17: Update Profile** | This use case describes the process by which a user updates their personal information and avatar. |
| **UC18: Promote Property** | This use case describes how a Property Owner pays to increase the ranking and visibility of their listing. |
| **UC19: Archive Property** | This use case describes the process by which a Property Owner or Administrator soft-deletes a property listing from the platform. |
| **UC20: Assign Agent to Property** | This use case describes how an Administrator assigns a Sales Agent to manage a specific property listing. |
| **UC21: Fetch Location Data** | This use case describes the retrieval of hierarchical location data (Provinces, Districts, Wards) for use in forms and filters. |
| **UC22: Wishlist Property** | This use case describes how a customer can add or remove a property from their personal favorites list for later review. |
| **UC23: Token Refresh** | This use case describes the process by which the system issues a new set of authentication tokens using a valid refresh token. |
| **UC24: Real-time Agent Chat** | This use case describes the real-time exchange of messages between a Customer and a Sales Agent regarding a property listing. |
| **UC25: AI Property Valuation** | This use case describes how the system uses artificial intelligence to estimate the market value of a property based on its features and historical data. |
| **UC26: Augmented Reality Virtual Tour** | This use case describes the process of initializing an Augmented Reality (AR) session for a 3D visualization of a property. |
| **UC27: Map-based Search** | This use case describes how a user can find properties by drawing an area on a map or navigating the map view. |
| **UC28: Escrow Protection** | This use case describes the process by which high-value transaction funds are held in a system-controlled virtual wallet until specific legal conditions are met. |
| **UC29: Referral Program** | This use case describes how the system tracks user invitations and awards reward points upon successful transactions by referred users. |
| **UC30: Agent Review** | This use case describes the process by which a customer rates and provides feedback on a Sales Agent following a completed interaction. |
| **UC31: View Owner Insights** | This use case describes how a Property Owner can view engagement metrics and conversion statistics for their listings. |
| **UC32: Scan Document with OCR** | This use case describes the automated extraction of text from identity or land documents to facilitate form completion. |
| **UC33: Subscribe to Plan** | This use case describes how a Sales Agent can purchase a premium subscription plan to increase their property listing capacity. |
| **UC34: Change Language** | This use case describes the process by which a user switches the application interface and content language. |
| **UC35: View Audit Logs** | This use case describes how an Administrator can view the history of sensitive system actions and state changes. |
| **UC36: Update Property** | This use case describes the process by which a Property Owner or Administrator updates the information and media of an existing property listing. |
| **UC37: Manage User Account** | This use case allows an Administrator to perform administrative actions on user accounts, including updating roles, statuses, or deleting the account. |
| **UC38: Manage Property Types** | This use case describes how an Administrator manages the global list of property categories (e.g., Apartment, Villa, Office). |
| **UC39: View User Activity History** | This use case describes how users can view a paginated list of their personal appointments, contracts, and payments. |
| **UC40: View Activity Details** | This use case describes the retrieval of full technical and legal details for a specific appointment or contract. |
